## Configuration

> [Describe](https://pigsty.io/docs/pgsql/config/) the cluster you want

- [Identity](https://pigsty.io/docs/pgsql/config/#identity): Parameters used for describing a PostgreSQL cluster
- [Primary](https://pigsty.io/docs/pgsql/config/#primary): Define a single instance cluster
- [Replica](https://pigsty.io/docs/pgsql/config/#replica): Define a basic HA cluster with one primary & one replica
- [Offline](https://pigsty.io/docs/pgsql/config/#offline): Define a dedicated instance for OLAP/ETL/Interactive queries.
- [Sync Standby](https://pigsty.io/docs/pgsql/config/#sync-standby): Enable synchronous commit to ensure no data loss
- [Quorum Commit](https://pigsty.io/docs/pgsql/config/#quorum-commit): Use quorum sync commit for an even higher consistency level
- [Standby Cluster](https://pigsty.io/docs/pgsql/config/#standby-cluster): Clone an existing cluster and follow it
- [Delayed Cluster](https://pigsty.io/docs/pgsql/config/#delayed-cluster): Clone an existing cluster for emergency data recovery
- [Citus Cluster](https://pigsty.io/docs/pgsql/config/#citus-cluster): Define a Citus distributed database cluster
- [Major Version](https://pigsty.io/docs/pgsql/config/#major-version): Define a PostgreSQL cluster with specific major version

## Playbook

> Materialize the cluster with idempotent [playbooks](https://pigsty.io/docs/pgsql/playbook/)

- [`pgsql.yml`](https://pigsty.io/docs/pgsql/playbook/#pgsqlyml) : Init HA PostgreSQL clusters or add new replicas.
- [`pgsql-rm.yml`](https://pigsty.io/docs/pgsql/playbook/#pgsql-rmyml) : Remove PostgreSQL cluster, or remove replicas
- [`pgsql-user.yml`](https://pigsty.io/docs/pgsql/playbook/#pgsql-useryml) : Add new business user to existing PostgreSQL cluster
- [`pgsql-db.yml`](https://pigsty.io/docs/pgsql/playbook/#pgsql-dbyml) : Add new business database to existing PostgreSQL cluster
- [`pgsql-monitor.yml`](https://pigsty.io/docs/pgsql/playbook/#pgsql-monitoryml) : Monitor remote PostgreSQL instance with local exporters
- [`pgsql-migration.yml`](https://pigsty.io/docs/pgsql/playbook/#pgsql-migrationyml) : Generate Migration manual & scripts for existing PostgreSQL
